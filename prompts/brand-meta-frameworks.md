# Brand Meta-Frameworks

Use this file to steer the agent like a design system strategist, not a procedural operator.

## 1. Brand truth vs presentation truth

Brand truth:
- exact mark anatomy
- palette
- typography voice
- product truth
- tone / vibe

Presentation truth:
- crop
- layout
- whitespace
- system framing
- motion attitude
- application style

Rule:
- keep brand truth fixed unless explicit redesign is requested
- let presentation truth move much more freely

## 2. Reference-role packs

Every serious run should classify references by role:
- composition
- motif
- application
- motion

Do not treat all references as one generic moodboard.

Use the pack asymmetrically:
- **primary roles** should shape the result
- **support roles** should only keep it honest

Most successful runs use:
- 1 brand-truth anchor
- 1 primary composition ref
- 1 primary motif or application ref

Anything beyond that should be a deliberate exception, not a default.

## 2b. Translate inspiration into mechanics

Agency inspiration is useful when it is translated into:
- hierarchy
- carrier logic
- family logic
- application attitude
- motion pacing

Do not borrow:
- their logo
- their wordmark
- their typography
- their copy
- their literal identity system

If a reference is so strong that the output starts looking like that agency's client, the reference was not translated enough.

## 3. Preserve / push

Before generating, decide:
- preserve: what cannot change
- push: what can be explored
- ban: what must not appear

## 4. Still before motion

Motion inherits the quality of the still.
If the still is weak, fix the still first.

## 5. Deterministic vs generative split

Use deterministic composition when you need:
- exact copy
- exact logo treatment
- poster boards
- sticker sheets
- layout precision

Use generative models when you need:
- exploration
- atmosphere
- alternate packaging
- graphic-system experimentation
- motion from approved stills

## 6. Concrete prompt materials

Good prompt materials are usually:
- exact mark anatomy
- approved primitives
- reference-role pack summary
- exact output count or scope
- one chosen system mechanic
- one composition structure
- negative constraints

Bad prompt materials are usually:
- mood-only adjectives
- too many style names
- vague words like `premium` without form nouns
- no explicit preserve / push split
- references with no clear role
- more references than the model can meaningfully weigh

## 7. One mechanic, many outputs

Strong identity systems usually work because one mechanic is repeated intelligently.

Examples:
- one frame move
- one band system
- one routed-line logic
- one crop language
- one reveal attitude in motion

If the output uses five unrelated tricks, it will not feel like a brand system.

## 8. Material sets, not one-offs

A brand run is stronger when it becomes a set:
- one hero or core explainer
- one product proof visual
- one social surface
- one motion surface
- one supporting system asset

Every material in the set should have:
- a job
- a target surface
- a product-truth expression
- a brand-anchor rule

Every material should either:
- show the stored logo / wordmark, or
- satisfy a clearly-branded anchor rule using palette, mark geometry, product truth, name, and approved carriers
