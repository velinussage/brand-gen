# Material Set Architecture

A material set is a **coherent family of branded outputs**, not a random batch of images.

## Formula

brand truth + product truth + translated inspiration mechanics = material set

## Translate inspiration, don't copy it

External references should be translated into:
- composition mechanics
- motif or family logic
- application attitude
- motion pacing

External references should **not** replace:
- the logo or wordmark
- brand typography
- product naming
- product UI truth
- exact mark geometry

## Material-set rules

Every set should define:
- goal
- primary surface
- set brand rule
- inspiration translation rule
- 4-5 materials with distinct jobs

Every material should define:
- job / purpose
- target surface
- product-truth expression
- abstraction level
- brand-anchor rule
- explicit plan path

## Branding rule

A material belongs in the set only if:
- the stored logo / wordmark is visible, **or**
- the material passes a clearly-branded anchor rule

A clearly-branded material should usually include at least 3 of:
- exact brand palette
- exact mark geometry or approved motif
- real product proof
- brand name or approved phrase
- approved carrier or composition pattern

## Default set order for product brands

1. landing hero
2. product proof visual
3. social launch visual
4. motion proof / bumper
5. supporting pattern or system asset

## Agent stance

The agent should ask:
- what is this set trying to explain or make people feel?
- what product truth must stay visible?
- what mechanic is worth translating from inspiration?
- is this still clearly the brand?
- does this material belong with the others?
