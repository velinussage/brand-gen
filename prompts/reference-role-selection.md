# Reference-Role Selection

Use this to choose quality refs before generation.

## Goal
Pick references that each teach **one thing well**.

## Roles

### Composition
Use for:
- hierarchy
- framing
- whitespace
- crop depth
- poster or page authority

Good composition refs:
- one dominant frame move
- obvious safe areas
- strong negative space

Bad composition refs:
- portfolio walls
- grids of unrelated case studies
- noisy collages

### Motif
Use for:
- repeat logic
- badge logic
- pattern behavior
- how one brand primitive becomes a family

Good motif refs:
- one primitive used several ways
- repeatable tile or band logic
- consistent line weight

Bad motif refs:
- one-off hero illustration
- abstract decoration with no system

### Application
Use for:
- posters
- merch
- stickers
- signage
- campaign surfaces

Good application refs:
- one coherent artifact family
- strong brand landing on a real surface

Bad application refs:
- general agency homepages
- too many artifact types at once

### Motion
Use for:
- reveal pacing
- mask logic
- edge wipes
- confident final hold

Good motion refs:
- one clear reveal mechanic
- 4–6 second pacing
- stable end frame

Bad motion refs:
- chaotic camera flights
- several reveal mechanics mixed together

## Selection heuristics

- pick the **tightest case-study page**, not the studio homepage
- prefer **one great ref per role** over several average refs
- if the artifact is flat, use **flat refs**
- if the artifact is a sheet, use **sheet refs**
- if the artifact is motion, choose refs with a **single motion idea**

## Simple rule

If you cannot explain in one short sentence why a ref was chosen, it should not be in the pack.
