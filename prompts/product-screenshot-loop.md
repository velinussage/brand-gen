# Product Screenshot Loop

Use this before capturing product screenshots.

## Ask first
1. Which product screens matter most right now?
2. Which states should be shown: default, filled, empty, modal, onboarding, comparison?
3. Do we need public marketing pages, authenticated product views, or both?
4. Which outputs will use these captures: browser illustration, banner, X card, X feed post, LinkedIn card, LinkedIn feed post, feature animation?
5. Do we want one wide contextual shot, one tighter feature shot, or both?

## Recommended shot types
- Homepage hero
- Dashboard overview
- Primary feature focus
- Detail / drilldown state
- Settings / workflow proof
- Mobile or narrow-state crop
- One “best branded view” for social cards

## Capture loop
1. Generate the shot list first.
2. Ask the user which shots to keep.
3. Capture only the approved URLs/views.
4. Review the captures with the user.
5. Gather 3–5 external presentation references from official product/marketing pages.
6. Write down what those references control: crop, framing, whitespace, background, depth.
7. Use the chosen captures as **product truth** and the external examples as **presentation references** for browser illustrations, product banners, social cards, and feed posts.

## Critical warning
- Do not ask the model to redesign the app unless the user explicitly wants a redesign.
- Do not let the model invent product copy, labels, cards, or fake features.
- Prefer a tighter crop on the strongest real product component over a full-window screenshot.
