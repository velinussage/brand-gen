# Material Type Differentiation

Use this before generating mixed brand-material batches.

The goal is to stop every output from collapsing into:
- warm neutral background
- one floating white card
- one smaller inset card
- gentle shadow
- frontal crop

If multiple material types look interchangeable, the prompt system is failing.

## First rule

Each material type needs its own:
- **job**
- **camera**
- **brand field**
- **graphic device**
- **surface count**

Change at least 3 of those between material types.

## Convergence traps

Do not let every batch default to:
- the same cream or beige field
- the same front-facing floating-card composition
- the same one-big-card-plus-one-small-card layout
- the same soft shadow system
- the same screenshot crop depth
- the same “editorial product marketing image” phrasing

If a new type looks like the previous type with a different aspect ratio, reject it.

## Role sheet

Before generation, write one line for each type:

- **browser-illustration** = faithful product section visual
- **product-banner** = wide atmospheric support visual
- **feature-illustration** = concept-led feature story visual
- **styleframe** = motion-ready campaign still
- **social / x-feed / x-feed-portrait / linkedin-feed-square** = mobile/poster-first promo visual
- **landing-hero** = deterministic page composition with exact copy/logo/nav

Then specify:
- hero product moment
- whether browser chrome is allowed
- supporting inset count
- background / field treatment
- one graphic device

## Material rules

### Browser illustration
- **Job:** show a real product moment clearly
- **Camera:** frontal or lightly cropped; readable UI first
- **Brand field:** quiet, supportive
- **Graphic device:** minimal frame, subtle edge, restrained depth
- **Surface count:** one hero window, inset optional
- **Avoid:** poster drama, fake campaign typography, too much environment

### Product banner
- **Job:** support a homepage section or campaign section
- **Camera:** wider, more environmental, less literal
- **Brand field:** does real work; more present than in browser illustration
- **Graphic device:** large color block, band, gradient wash, mask, or spatial stage
- **Surface count:** often one hero crop only; inset is optional, not default
- **Avoid:** looking like a screenshot board or compare sheet

### Feature illustration
- **Job:** explain one capability or workflow
- **Camera:** tighter on the product proof
- **Brand field:** bridges UI and concept
- **Graphic device:** one deliberate explanatory move: crop mask, connector, spotlight, sequence cue, or diagrammatic shape
- **Surface count:** one hero crop, one support detail max
- **Avoid:** generic floating card with no concept or message

### Styleframe
- **Job:** feel like a motion-ready campaign still
- **Camera:** bolder crop or more cinematic spatial tension
- **Brand field:** most expressive of the still-image types
- **Graphic device:** lighting direction, large brand geometry, layered depth, directional motion cue
- **Surface count:** zero or one support crop
- **Avoid:** same layout as browser illustration with a stronger shadow

### Social square / feed / portrait
- **Job:** stop the scroll and stay readable on mobile
- **Camera:** poster-like, closer, simpler
- **Brand field:** stronger contrast and clearer silhouette
- **Graphic device:** bold crop, caption-safe void, single frame lockup, or strong border system
- **Surface count:** usually one crop only
- **Avoid:** mini landing page, tiny text, multiple equal panels

## Distinctive graphic devices

Choose one per asset:
- oversized crop against a field
- edge bleed
- hard frame with deep margin
- masked crop inside a shape
- split-field composition
- poster border
- directional gradient wash
- motion streak / reveal band
- spotlight / vignette
- repeated micro detail strip

Do not stack several unless the composition truly needs it.

## Quick checks

Ask before spending credits:
1. What makes this material type visually different from the previous one?
2. What would still read if the image were seen at phone size?
3. Is the brand field doing enough, or is the UI doing all the work?
4. Did we default to the same floating-card composition again?

## Material DNA lock

Before rendering, declare:
- TYPE
- PURPOSE
- EMOTION
- COMPOSITION
- PALETTE MODE
- TYPO VOICE

If too many of those match the previous output, mutate before generation.

## Distinctness gate

For one batch, each material should be visually separable at thumbnail size.
Require at least 3 differences across:
- color energy
- framing
- typography
- content density

If that test fails, regenerate with a different structure first and style second.
